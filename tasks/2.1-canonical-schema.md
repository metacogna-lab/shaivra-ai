## Objective

Define and implement the canonical intelligence schema that ALL OSINT signals must normalize to before entering the knowledge graph. This schema provides a unified data model for entities, observations, relationships, and intelligence events.

## Implementation Steps

### 1. Create TypeScript Type Definitions

**File:** `src/contracts/intelligence.ts`

Define the following core types:

#### EntityReference
Represents a discovered entity (person, organization, infrastructure) with confidence scoring.

```typescript
export interface EntityReference {
  id: string;                    // Unique identifier (UUID)
  type: 'person' | 'organization' | 'infrastructure' | 'event' | 'unknown';
  name: string;                  // Primary name/identifier
  aliases: string[];             // Alternative names
  confidence: number;            // Confidence score 0.0-1.0
  attributes: Record<string, any>; // Flexible attributes
  sourceIds: string[];           // References to source observations
  firstSeen: Date;
  lastSeen: Date;
  metadata: {
    verified: boolean;
    tags: string[];
    notes?: string;
  };
}
```

#### Observation
Represents a specific attribute, behavior, or event associated with an entity.

```typescript
export interface Observation {
  id: string;                    // Unique identifier (UUID)
  entityId: string;              // Reference to EntityReference
  type: 'attribute' | 'behavior' | 'event' | 'relationship';
  property: string;              // What was observed (e.g., "email", "location", "post")
  value: any;                    // The observed value
  confidence: number;            // Confidence score 0.0-1.0
  source: {
    tool: string;                // Which OSINT tool produced this
    url?: string;                // Source URL if applicable
    timestamp: Date;             // When observation was made
    raw: any;                    // Raw data from tool
  };
  context: Record<string, any>;  // Additional context
  expiresAt?: Date;              // When observation becomes stale
}
```

#### Relationship
Represents a connection between two entities with evidence tracking.

```typescript
export interface Relationship {
  id: string;                    // Unique identifier (UUID)
  fromEntityId: string;          // Source entity
  toEntityId: string;            // Target entity
  type: string;                  // Relationship type (e.g., "employs", "located_in", "mentions")
  strength: number;              // Relationship strength 0.0-1.0
  confidence: number;            // Confidence score 0.0-1.0
  evidence: string[];            // Observation IDs supporting this relationship
  bidirectional: boolean;        // Is relationship symmetric?
  metadata: {
    firstSeen: Date;
    lastSeen: Date;
    count: number;               // How many times observed
    context?: string;
  };
}
```

#### IntelligenceEvent
Wrapper for entities, observations, and relationships from a single OSINT tool run.

```typescript
export interface IntelligenceEvent {
  id: string;                    // Unique identifier (UUID)
  traceId: string;               // Links to investigation
  investigationId?: string;      // Optional investigation FK
  tool: string;                  // Which OSINT tool generated this
  target: string;                // Investigation target
  timestamp: Date;
  status: 'success' | 'partial' | 'failed';
  entities: EntityReference[];
  observations: Observation[];
  relationships: Relationship[];
  metadata: {
    executionTime: number;       // Milliseconds
    cost?: number;               // API cost if applicable
    errors?: string[];
    raw?: any;                   // Raw tool output (for debugging)
  };
}
```

### 2. Add Prisma Database Models

**File:** `prisma/schema.prisma`

Add the following models (append to existing schema):

```prisma
model IntelligenceEvent {
  id              String   @id @default(uuid())
  traceId         String   @map("trace_id")
  investigationId String?  @map("investigation_id")
  tool            String
  target          String
  timestamp       DateTime @default(now())
  status          String   // 'success' | 'partial' | 'failed'
  entities        Json     // EntityReference[]
  observations    Json     // Observation[]
  relationships   Json     // Relationship[]
  metadata        Json     // { executionTime, cost, errors, raw }
  createdAt       DateTime @default(now()) @map("created_at")
  updatedAt       DateTime @updatedAt @map("updated_at")

  investigation   Investigation? @relation(fields: [investigationId], references: [id], onDelete: Cascade)

  @@index([investigationId])
  @@index([traceId])
  @@index([timestamp])
  @@index([tool])
  @@map("intelligence_events")
}

model TaskQueue {
  id            String   @id @default(uuid())
  taskType      String   @map("task_type")
  payload       Json
  priority      Int      @default(5) // 1-10, higher = more urgent
  status        String   @default("pending") // 'pending' | 'running' | 'completed' | 'failed'
  attempts      Int      @default(0)
  maxAttempts   Int      @default(3) @map("max_attempts")
  result        Json?
  error         String?
  createdAt     DateTime @default(now()) @map("created_at")
  startedAt     DateTime? @map("started_at")
  completedAt   DateTime? @map("completed_at")
  updatedAt     DateTime @updatedAt @map("updated_at")

  @@index([status, priority])
  @@index([taskType])
  @@index([createdAt])
  @@map("task_queue")
}
```
