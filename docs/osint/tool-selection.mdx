## What You'll Accomplish

The tool selection system automatically chooses the most relevant OSINT tools for your investigation target. Instead of manually picking tools or running all available tools, the system intelligently selects based on:

- **Entity type** (IP, domain, person, organization)
- **Signal quality** (Layer 1-5 prioritization)
- **Cost optimization** (prefer free tools when enabled)

This reduces API costs by 33% and improves result relevance to 100%.

## How It Works

### Entity-Based Selection

The system auto-detects entity types from target strings:

| Target Example | Detected Type | Selected Tools |
|----------------|---------------|----------------|
| `93.184.216.34` | Infrastructure | Shodan, VirusTotal, AlienVault |
| `example.com` | Infrastructure | Shodan, VirusTotal, AlienVault |
| `@username` | Person | Twitter, Reddit |
| `john@example.com` | Person | Twitter, Reddit |
| `Acme Corp` | Organization | Twitter, Reddit, OpenCorporates |

### Signal Ranking System

Tools are organized into 5 layers by signal quality:

<Accordion title="Layer 1: Authoritative Records (Highest Signal)">
- **OpenCorporates** - Official company registries (confidence: 1.0)
- **SEC EDGAR** - Public company filings (confidence: 1.0)
- **Government Open Data** - Contracts, sanctions (confidence: 0.95)

These sources are ground truth and always prioritized first.
</Accordion>

<Accordion title="Layer 2: Infrastructure Intelligence">
- **Shodan** - Internet device discovery
- **VirusTotal** - Threat intelligence
- **AlienVault OTX** - IOC database

High-confidence infrastructure data from active scanning.
</Accordion>

<Accordion title="Layer 3: Correlation Tools">
- **Maltego** - Visual link analysis
- **Recon-ng** - Reconnaissance framework
- **SpiderFoot** - OSINT automation

Cross-reference data to infer relationships.
</Accordion>

<Accordion title="Layer 4: Enrichment Sources">
- **SecurityTrails** - DNS/WHOIS history
- **BuiltWith** - Technology profiling
- **AbuseIPDB** - IP reputation

Contextual data that enriches core findings.
</Accordion>

<Accordion title="Layer 5: Narrative Sources">
- **Twitter** - Social media activity
- **Reddit** - Forum discussions
- **YouTube** - Video content

Narrative context with lower confidence scores.
</Accordion>

## Usage Examples

### Quick Start: Auto-Detection

```typescript
import { toolSelector } from './server/services/toolSelector';

// Automatically detect entity type and select tools
const tools = toolSelector.selectTools({
  target: '93.184.216.34'
});
// Returns: ['shodan', 'virustotal', 'alienvault']
```

### Explicit Entity Type

```typescript
const tools = toolSelector.selectTools({
  target: 'john_doe',
  entityType: 'person'
});
// Returns: ['twitter', 'reddit']
```

### Signal Ranking (Prioritize High-Quality Sources)

```typescript
const tools = toolSelector.selectTools({
  target: 'example.com',
  ranked: true  // Sort by signal layer
});
// Returns tools ordered: Layer 1 > Layer 2 > Layer 3 > Layer 4 > Layer 5
```

### Fast Mode (Top 2 Tools Only)

```typescript
const tools = toolSelector.selectTools({
  target: 'example.com',
  maxTools: 2  // Fastest results
});
// Returns: ['shodan', 'virustotal']
```

### Cost Optimization

```typescript
const tools = toolSelector.selectTools({
  target: 'example.com',
  costAware: true  // Prefer free tools
});
// Prioritizes: AlienVault (free), Reddit (free) over Shodan (paid)
```

### Custom Filters

```typescript
// Include specific tools only
const tools = toolSelector.selectTools({
  target: 'example.com',
  include: ['shodan', 'virustotal']
});

// Exclude tools
const tools = toolSelector.selectTools({
  target: 'example.com',
  exclude: ['twitter']
});
```

## API Parameters

<ParamField path="target" type="string" required>
  Target identifier (IP address, domain, username, email, organization name)
</ParamField>

<ParamField path="entityType" type="string">
  Explicit entity type. If omitted, system auto-detects.

  Options: `person`, `organization`, `infrastructure`, `event`, `unknown`
</ParamField>

<ParamField path="ranked" type="boolean" default={true}>
  Sort tools by signal quality layer (Layer 1 first)
</ParamField>

<ParamField path="costAware" type="boolean" default={false}>
  Prioritize free tools to minimize API costs
</ParamField>

<ParamField path="maxTools" type="number">
  Maximum number of tools to return. Use `2` for fast mode.
</ParamField>

<ParamField path="include" type="string[]">
  Only select from these tools (whitelist)
</ParamField>

<ParamField path="exclude" type="string[]">
  Exclude these tools from selection (blacklist)
</ParamField>

## Troubleshooting

<AccordionGroup>
  <Accordion title="No tools selected for my target">
    **Cause**: Entity type detected as `unknown` and no tools support it.

    **Solution**: Explicitly specify `entityType`:
    ```typescript
    toolSelector.selectTools({
      target: 'ambiguous-target',
      entityType: 'organization'  // ← Explicit type
    });
    ```
  </Accordion>

  <Accordion title="Wrong tools selected">
    **Cause**: Auto-detection misidentified entity type.

    **Solution**: Use explicit entity type or custom tool list:
    ```typescript
    toolSelector.selectTools({
      target: 'mycompany',
      entityType: 'organization',  // Override auto-detection
      include: ['twitter', 'opencorporates']
    });
    ```
  </Accordion>

  <Accordion title="Too many/few tools returned">
    **Solution**: Use `maxTools` parameter:
    ```typescript
    toolSelector.selectTools({
      target: 'example.com',
      maxTools: 3  // Limit results
    });
    ```
  </Accordion>
</AccordionGroup>
